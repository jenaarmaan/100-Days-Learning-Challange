## Topic: C21. Classical Planning Problems

---

## 1. One-line definition (in your own words)

**A Classical Planning Problem is a search problem where an agent must find a sequence of actions that transforms an initial state into a goal state under deterministic, fully observable conditions.**

---

## 2. Problem it solves

### Why this exists

In many AI tasks, we don’t just want a single decision — we want a *sequence of actions* that achieves a goal.

Examples:

- Robot moving objects
- Game level solving
- Logistics routing
- Automated task execution

Classical planning provides a structured framework to compute:

> A valid action sequence that guarantees goal achievement.

---

### What fails without it

- Greedy decision-making may fail long-term.
- Hard to reason about multi-step consequences.
- Cannot formally verify if a goal is reachable.

---

## 3. Core idea (intuition)

A classical planning problem consists of:

```text
1. Initial State
2. Goal State
3. Set of Actions
```

Each action has:

- Preconditions
- Effects (Add list, Delete list)

Planning becomes:

> Searching in the space of states generated by applying actions.

---

### Example: Simple Robot World

Initial:

```text
Robot at A
Box at B
```

Goal:

```text
Box at A
```

Actions:

```text
Move(A → B)
Pick(Box)
Move(B → A)
Drop(Box)
```

Planner must determine correct order.

---

## 4. How it works (high-level steps)

### Step 1

Represent world as logical state (set of facts).

### Step 2

Define actions with:

- Preconditions
- Add effects
- Delete effects

### Step 3

Search over state space:

- Apply valid actions
- Generate successor states

### Step 4

Stop when goal condition satisfied.

---

## 5. Strengths

- Clean formal model.
- Deterministic guarantees.
- Suitable for symbolic AI.
- Foundational for robotics and automation.

---

## 6. Weaknesses / failure cases

- State space explosion.
- Assumes full observability.
- Assumes deterministic actions.
- Not suitable for uncertainty.

---

## 7. Where it is used in real systems

- Robotics task planning
- Automated manufacturing
- Logistics optimization
- Game AI
- Workflow automation

Modern AI systems extend classical planning with uncertainty and learning.

---

## 8. Keywords / terms to remember

- Initial state
- Goal state
- Action model
- Preconditions
- Add effects
- Delete effects
- State space search
- Deterministic planning
